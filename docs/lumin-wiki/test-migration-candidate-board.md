# Test Migration Candidate Board

This page decides which test suite may be reviewed for the next Vitest pilot.
It is not a migration plan by itself. A suite may move only after a review names
the invariant, the edge-case failure, the preserved Node command, the focused
Vitest command, and the fixture boundary.

## Status Vocabulary

- `READY-FOR-REVIEW`: small enough to write a pilot review next.
- `REVIEW-FIRST`: plausible later, but the invariant or boundary needs a review
  page before code changes.
- `REVIEWED`: review page exists; implementation may proceed in a narrow PR
  that keeps the preserved Node command.
- `PARKED`: do not migrate yet; the suite protects analyzer semantics where a
  runner rewrite could hide evidence drift.
- `DONE`: row-level status only. The focused Vitest mirror exists and has been
  reviewed; the wiki remains living documentation and must keep being updated.
