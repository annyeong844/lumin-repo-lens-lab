# Lumin Work Tracker

> **Role:** maintainer-facing tracker for long-running Lumin Repo Lens work.
> **Status:** living checklist.
> **Last updated:** 2026-06-01

This tracker exists to prevent small MVP slices from being mistaken for complete
features. A merged PR may move an item to `MVP`, but an item reaches `DONE` only
when the listed acceptance checks are satisfied and the public skill/package has
been verified when user-visible behavior changed.

Status vocabulary:

- `SPEC`: design captured, no behavior change expected.
- `NEXT`: next implementation candidate.
- `MVP`: useful slice merged, but documented follow-up remains.
- `VERIFY`: implementation merged; install or corpus verification still needed.
- `DONE`: acceptance checks complete.
- `PARKED`: intentionally not active.

## Completion Rules

- Do not mark an item `DONE` from a passing unit test alone if the behavior is
  user-visible and the package has not been installed or dogfooded.
- Do not mark an item `DONE` when the implementation is intentionally narrow.
  Use `MVP` and spell out the missing acceptance check.
- Do not add broad abstractions just to make this table shorter. If two rows
  share a pattern but have different proof contracts, keep them separate.
- If a PR only bumps the package version, update this tracker only when the bump
  changes verification state.
