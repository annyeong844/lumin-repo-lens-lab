## 4. canonical/canon-drift.md v1 — full structure

Each sub-section below corresponds to a section of the canonical file. The actual `.md` content lands verbatim in `canonical/canon-drift.md` during step 1; this is the reviewable draft.

### 4.1 §1 Purpose

One paragraph: stickiness loop step 4. Skill emits drift evidence; human/LLM decides re-promotion; skill NEVER writes to `canonical/`. Cross-ref to `docs/history/phases/p5/session.md` v2 §2 "Optimization target".

### 4.2 §2 Drift fact kinds

4 kinds, one per P3 source:
- `type-drift` — consumes `canonical/type-ownership.md` + fresh `collectTypeIdentities`.
- `helper-drift` — consumes `canonical/helper-registry.md` + fresh `collectHelperIdentities`.
- `topology-drift` — consumes `canonical/topology.md` + fresh `collectTopologyStructure`.
- `naming-drift` — consumes `canonical/naming.md` + fresh `collectNamingCohorts`.

### 4.3 §3 Drift categories + family tags

**20 source-specific categories** (enum per kind):

```
type-drift:
  identity-added    (family: added)
  identity-removed  (family: removed)
  label-changed     (family: label-changed)
  owner-changed     (family: structural-status-changed)

helper-drift:
  helper-added           (family: added)
  helper-removed         (family: removed)
  label-changed          (family: label-changed)
  contamination-changed  (family: content-shifted)
  fan-in-tier-changed    (family: label-changed)

topology-drift:
  submodule-added     (family: added)
  submodule-removed   (family: removed)
  scc-status-changed  (family: structural-status-changed)
  oversize-changed    (family: content-shifted)
  cross-edge-added    (family: added)
  cross-edge-removed  (family: removed)

naming-drift:
  cohort-added               (family: added)
  cohort-removed             (family: removed)
  cohort-convention-shifted  (family: label-changed)
  new-outlier-introduced     (family: content-shifted)
  outlier-resolved           (family: content-shifted)
```

**5 family tags** (generic grouping):

```
added
removed
label-changed
structural-status-changed
content-shifted
```

Every category MUST map to exactly one family. Test extension pins this mapping.

### 4.4 §4 Identity contract

Drift records key on the same identity format each source's canon uses:
- `type-drift` / `helper-drift` → `ownerFile::exportedName` (matches `canonical/identity-and-alias.md` §2).
- `topology-drift` submodule-level categories → `<submodule>` path string.
- `topology-drift` cross-edge categories → `<from-submodule> → <to-submodule>` edge label.
- `topology-drift` oversize-changed → `<ownerFile>` path.
- `naming-drift` cohort-level → `<submodule>` (file cohort) or `<submodule>::<kind>` (symbol cohort, kind ∈ `{type-export, helper-export, constant-export}`).
- `naming-drift` outlier categories → per-item identity (file item `<ownerFile>` or symbol item `<ownerFile>::<exportedName>`).

### 4.5 §5 Parser contract

Per-source table shape that each P3 renderer commits to. Parser is allowed strict mode on these exact shapes:

**5.a `type-ownership.md`** — from `_lib/canon-draft-types.mjs::renderTypeOwnership`.
- Required columns (ordered): `Name`, `Identity`, `Owner`, `Fan-in`, `Status`, `Tags`.
- Identity cell extract: strip backticks, expect `ownerFile::exportedName` format.
- Status cell extract: first whitespace-separated token ∈ canonical §9 label set.

**5.b `helper-registry.md`** — from `renderHelperRegistry`.
- Required columns: `Name`, `Identity`, `Owner`, `Signature`, `Fan-in`, `Status`, `Tags`, `Any / unknown signal`.
- Status cell: first token ∈ canonical §10.3 label set.

**5.c `topology.md`** — from `renderTopology` (multi-section):
- §1 Submodule inventory: columns `Submodule | Files | LOC | In-edges | Out-edges | SCC | Status | Tags`. Status ∈ canonical §11.1 label set.
- §3 Cycle listing: headline `### Cycle <N> (size <M>) — forbidden-cycle ❌` + bulleted member list.
- §4 Oversize files: columns `File | LOC | Status`. Status ∈ `{oversize, extreme-oversize}` per §11.3.
- §5 Workspace boundaries (monorepo only — may be absent): columns `Package | Path | Files | LOC`.

**5.d `naming.md`** — from `renderNaming` (multi-section):
- §1 File cohorts: columns `Cohort (submodule) | Files | DominantConvention | ConsistencyRate | OutliersCount | Status`. Status ∈ canonical §12.3 label set.
- §2 Symbol cohorts: columns `Cohort (submodule::kind) | Items | DominantConvention | ConsistencyRate | OutliersCount | Status`.
- §3 Outliers (optional — omitted when zero): columns `Identity | Cohort | Name | ObservedConvention | DominantConvention | Status`.

**Parser strictness policy (session v2 P0-2):**
- Unknown column / missing required column / unknown Status label → **`canon-parse-error`** diagnostic per source.
- Unrecognized first-row shape (header row doesn't match any known contract) → source-level **`unrecognized-canon-schema`** skip with `perSource[source].status = "skipped-unrecognized-schema"`.
- Lenient partial parse is **explicitly forbidden** — a drift detector that reads half the rows and reports "no drift" is the worst failure mode.

### 4.6 §6 JSON artifact shape

`<output>/canon-drift.json` minimal schema. P5-1+ MUST use this shape. Additive field additions allowed; removals / renames require canonical edit.

```json
{
  "meta": {
    "tool": "check-canon.mjs",
    "generated": "ISO-8601",
    "root": "/abs/path",
    "canonDir": "/abs/path/canonical",

    "strict": false
  },
  "summary": {
    "sourcesRequested": 4,
    "sourcesChecked": 2,

  },
  "perSource": {
    "type-ownership": {
      "status": "drift",
      "driftCount": 2,
      "reportPath": "<output>/canon-drift.type-ownership.md",
      "diagnostics": []
    },
    "helper-registry": {
      "status": "clean",
      "driftCount": 0,
      "reportPath": "<output>/canon-drift.helper-registry.md",
      "diagnostics": []
    },
    "topology": {
      "status": "skipped-missing-canon",
      "driftCount": 0,
      "diagnostics": [{ "reason": "canonical/topology.md absent" }]
    },
    "naming": {
      "status": "skipped-unrecognized-schema",
      "driftCount": 0,
      "diagnostics": [{ "reason": "first-row header not in parser contract" }]
    }
  },
  "drifts": [
    {
      "kind": "type-drift",
      "category": "owner-changed",
      "family": "structural-status-changed",
      "identity": "src/foo.ts::User",
      "canon": {
        "file": "canonical/type-ownership.md",
        "line": 42,
        "label": "single-owner-strong",
        "owner": "src/foo.ts:14"
      },
      "fresh": {
        "label": "single-owner-strong",
        "owner": "src/types/user.ts:8"
      },
      "confidence": "high"
    }
  ]
}
```

**`perSource[source].status`** domain: `drift` / `clean` / `skipped-missing-canon` / `skipped-unrecognized-schema` / `parse-error`.
