## 2. Scope

### 2.1 In (P5-1)

- **`_lib/check-canon-utils.mjs`** — **pure** parser + drift-record primitives (no fs I/O):
  - `parseTypeOwnershipCanonText({ text, labelSet })` — parses a string, returns records + diagnostics + status. Never reads files.
  - `makeDriftRecord({ kind, category, identity, canon, fresh })` — auto-derives `family` from canon-drift.md §3.1 mapping.
  - `buildCanonDriftJsonObject({ meta, perSource, drifts })` — constructs the canon-drift.md §6 JSON object in memory. Does NOT write.
  - Diagnostic helpers (category tags: `canon-parse-error`, `skipped-missing-canon`, `skipped-unrecognized-schema`).
- **`_lib/check-canon-artifact.mjs`** — I/O layer (separate from utils so purity is not mixed):
  - `loadTypeOwnershipCanon({ canonPath })` — wraps `parseTypeOwnershipCanonText` with fs read + missing-file detection. Returns `{ status: 'skipped-missing-canon' }` when absent.
  - `writeCanonDriftArtifacts({ output, driftObject, reportMarkdown, source })` — atomic write of `canon-drift.json` + `canon-drift.<source>.md`. No append-merge: each single-source invocation writes a fresh JSON containing only the requested `perSource` entry. `--source all` aggregation (P5-4) will extend this writer with a single-invocation multi-source mode — still no cross-invocation merge.
- **`_lib/check-canon-types.mjs`** — type-ownership drift engine:
  - Parser: reads `canonical/type-ownership.md` via utils parser. Returns `Map<identity, CanonRow>` + diagnostics.
  - Fresh-record collector: calls `collectTypeIdentities` (P3-1 reuse) + emits the same `Map<identity, FreshRow>` shape.
  - Differ: set-diff + pairwise label/owner compare. Upgrades "removed X@A + added X@B" → `owner-changed X@A→B` via a name-based post-pass.
  - Renderer: per-category tables in `canon-drift.type-ownership.md`.
- **`check-canon.mjs`** — CLI at repo root. v1 accepts `--source type-ownership` only; other values → exit 2 with `[check-canon] --source=X not yet implemented (P5-2/3/4).`
- **Fixtures** — `tests/fixtures/canon-drift-types-{clean,added,removed,label-changed,owner-changed}/` — one per category + one clean baseline.
- **Tests** — 5 new files per §6.

```

| Identity | Fresh owner | Fresh label |
|----------|-------------|-------------|

## 3. identity-removed

| Identity | Canon owner | Canon label | Canon line |
|----------|-------------|-------------|-----------:|

## 4. label-changed

| Identity | Canon label | Fresh label | Canon line |
|----------|-------------|-------------|-----------:|
| `<ownerFile>::<exportedName>` | `<label>` | `<label>` | `<N>` |

## 5. owner-changed

| Exported name | Canon owner | Fresh owner | Canon label | Fresh label | Canon line |

**Note on `--include-tests` / `--production` / `--exclude`:** these are *not* forwarded to check-canon. The current-observation collector consumes a pre-built `symbols.json`; scope flags belong to the producer that wrote `symbols.json`. If symbol scope matters for drift interpretation, re-run the producer (`build-symbol-graph.mjs` or `audit-repo.mjs --pre-write`) with the desired flags, then re-run check-canon.

**Standalone exit codes (per session §5.1 P0-1/P0-4 + canon-drift.md §5.e):**

- **0** — `perSource["type-ownership"].status === "clean"` AND no diagnostics.
- **1** — `driftCount > 0`. Drift detected.
- **2** — any of:
  - `status === "parse-error"` (unknown column, unknown Status token, etc.).
  - `status === "skipped-unrecognized-schema"` (header row doesn't match §5.a contract).
  - `status === "skipped-missing-canon"` (`canonical/type-ownership.md` absent). **Single-source runs always hard-fail on missing canon in v1** — this matches session §5.1 P0-4 "single-source → exit 2 on missing canon". `--source all`'s per-source-skip behavior arrives only in P5-4.
  - Missing `--source`.
  - `--output <dir>` missing or `symbols.json` absent from it.
  - Unreadable `--canon-dir`.
  - Unknown `--source` value (including `helper-registry` / `topology` / `naming` / `all` in v1).

Stdout: short status banner (`[check-canon] type-ownership: <N> drifts` or `[check-canon] type-ownership: clean` or `[check-canon] type-ownership: skipped (<reason>)`) + report path.
Stderr: `[check-canon]` diagnostic chatter when `--verbose` OR on error.

- `makeDriftRecord({ kind, category, identity, canon, fresh, confidence })` — constructs a record; auto-derives `family` from a frozen module-local `CATEGORY_TO_FAMILY` map (sourced from canon-drift.md §3.1 and pinned by DC-13 in P5-0's test-classification-gates).
- `buildCanonDriftJsonObject({ meta, perSource, drifts })` — returns the plain object shape per canon-drift.md §6. Never writes.

**`_lib/check-canon-artifact.mjs`** (~80 LOC, I/O layer):

- `loadTypeOwnershipCanon({ canonPath, canonLabelSet }) → { records, diagnostics, status, lineCount }`:
  - File absent → `{ status: 'skipped-missing-canon', diagnostics: [{reason: 'canon-file-absent'}], records: new Map(), lineCount: 0 }`.
  - File present → `readFileSync` + delegate to `parseTypeOwnershipCanonText`.
- `writeCanonDriftArtifacts({ output, driftObject, reportMarkdown, source }) → { jsonPath, reportPath | null }`:
  - Always writes `<output>/canon-drift.json` (overwrite, atomic temp+rename via `_lib/artifacts.mjs`).
  - Writes `<output>/canon-drift.<source>.md` only if `reportMarkdown` is non-null (callers pass null for skipped-missing-canon per §5.6).
  - **No append-merge** (reviewer P0-6). Each invocation writes a fresh JSON reflecting only the sources included in this run. P5-4 `--source all` will still be a single invocation writing all requested sources together; cross-invocation merge is explicitly out of scope.

**Exit:** ~26 assertions green (18 utils + 8 artifact).

### 5.3 Step 2 — `_lib/check-canon-types.mjs` engine

**Makes green:** `test-check-canon-types.mjs`.

**Impl:** ~180 LOC leaf module. Pure aside from its caller-supplied `loadTypeOwnershipCanon` (injectable for unit tests).

- `detectTypeOwnershipDrift({ canonPath, symbols, canonLabelSet, loader? }) → { drifts, status, diagnostics, reportMarkdown | null, canonLineCount }`:
  - Call `loader = loader ?? loadTypeOwnershipCanon` with `canonPath` + `canonLabelSet`. Any non-clean status from the loader (`skipped-missing-canon` / `skipped-unrecognized-schema` / `parse-error`) is returned verbatim with `drifts: []` and `reportMarkdown: null`.
  - Call `collectTypeIdentities({ symbols })` (from `_lib/canon-draft-types.mjs` — PF-4) for the current-observation side.
  - **Differ algorithm:**
    1. Build `canonByIdentity: Map<identity, CanonRow>` and `freshByIdentity: Map<identity, FreshRow>`.
    2. First pass (identity-based): for each key union, emit `identity-removed` (canon-only), `identity-added` (fresh-only), `label-changed` (both, differing label).
    3. Second pass (name-based owner-change upgrade): group added + removed by `exportedName`. If exactly one add + one remove share a name, delete both and emit one `owner-changed` carrying `canon.owner`, `fresh.owner`, `canon.label`, `fresh.label` (labels MAY be equal; they are preserved either way per reviewer P0-5).
    4. Two-or-more collisions on one `exportedName` (e.g. 2 added + 1 removed) → leave as separate added/removed records with `confidence: 'low'`. Pairing requires 1:1 for safety (R1 — rename-inference ambiguity).
  - Render `canon-drift.type-ownership.md` per §4.1 shape (§1 summary + per-category tables omitted when their count is 0). `reportMarkdown: null` when status is anything other than `drift` or `clean`.
- `renderTypeOwnershipDriftReport({ drifts, canonPath, canonLineCount })` — pure renderer.

**Exit:** ~25 assertions green.

## 8. Known risks

- **P5-1-R1: Rename-inference ambiguity.** When 2+ add/remove pairs share a single `exportedName`, pairing cannot be 1:1. **Mitigation:** v1 leaves them as separate added/removed records with `confidence: 'low'` hint; reviewer manually re-pairs via session v2 P1-9 "manual rename". v2+ upgrades via P4 shape-hash.
- **P5-1-R2: `symbols.json` freshness.** `collectTypeIdentities` takes a `symbols` input object — P5-1 loads `<output>/symbols.json`. If the user runs `check-canon.mjs` without first running `build-symbol-graph.mjs` directly (or `audit-repo.mjs --pre-write`, which orchestrates the symbol producer), `symbols.json` is absent → CLI exits 2 with guidance. If the file is present but stale (code changed since last run), drift results are stale too. **Mitigation:** stderr warning when `symbols.json` mtime is older than any watched `src/**/*.ts` mtime; hard-fail via a future `--strict-freshness` flag. Ship stale-warning in v1; the strict flag is deferred unless reviewer requests.
- **P5-1-R3: Parser-contract / renderer drift after P5-1 lands.** If a future P3-1 renderer patch shifts column headers, the canon-drift.md §5.a contract + check-canon parser both need to update. **Mitigation:** existing `tests/test-canon-drift-parser-contract.mjs` round-trip (P5-0) already fails on column drift; flagged reviewer pair-commit rule (renderer + §5 together). P5-1 adds NO new pin on top of P5-0's round-trip.
- **P5-1-R4: Current-observation freshness and cost.** P5-1 loads `<output>/symbols.json` and projects it via `collectTypeIdentities`; it does NOT re-extract ASTs. Cost is JSON load + in-memory projection — typically sub-second even on large repos. The real freshness risk is `symbols.json` being stale relative to source mtimes (see R2). **Mitigation:** stderr warning on stale `symbols.json` (R2); a true on-demand AST mode is v2+ and remains out of P5-1 scope.
- **P5-1-R5: `confidence` field semantics.** v1 uses `'high'` for crisp diffs + `'low'` for ambiguous rename pairs. v2+ may introduce a fuller tier. **Mitigation:** keep as `'high' | 'low'` string union; don't introduce numeric scoring until P4 shape-hash lands.
- **P5-1-R6: Single-source vs `--source all` exit-code asymmetry.** Session v2 P0-4 says single-source → hard fail on missing canon; `--source all` → per-source `skipped-missing-canon`. P5-1 only implements the single-source branch; P5-4 adds the aggregate branch. **Mitigation:** v1 explicitly rejects `--source all` so the asymmetry cannot surface mid-phase.
