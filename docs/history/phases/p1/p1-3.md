## 2. Scope

### 2.1 In (P1-3)

- **`_lib/pre-write-drift.mjs`** — `computeDrift(canonicalClaims, lookups)`. Given the parsed canonical owner claims (from P1-0 `pre-write-canonical-parser.mjs`) and the name lookup results (from P1-1 `pre-write-lookup-name.mjs`), emit one drift entry per disagreement. Drift entries land in `advisory.drift[]` and are rendered under a new `### Canonical drift` section with `CANONICAL DRIFT:` wording.
- **Cold-cache auto-run** in `pre-write.mjs` — **artifact-by-artifact preflight**. Each of `symbols.json` / `topology.json` / `triage.json` is checked independently; only the missing ones trigger their producer. `symbols.json` present + `topology.json` missing → only `measure-topology.mjs` is spawned. Opt-out via `--no-fresh-audit` flag; on by default. Failure / timeout of any cold-cache script emits `failures[{kind: 'cold-cache-<artifact>-failed|timeout', reason}]` and leaves that lookup path degraded, never crashing the CLI. **All producer spawns use `execFileSync` with argv arrays; `execSync` with shell strings is FORBIDDEN** — root/output paths may contain spaces, `$`, or parentheses, and shell-string execution has bitten this repo before.
- **`audit-repo.mjs --pre-write` opt-in flag** — when set, runs `pre-write.mjs` as a final step. Not in default `quick` / `full` profiles. The flag REQUIRES `--intent <file>`. If `--intent` is missing, `audit-repo.mjs` completes the rest of the pipeline, writes `manifest.json.preWrite = { requested: true, ran: false, reason: "--intent missing" }`, prints a helpful stderr message, AND **exits with code 2**. Rationale: a user who explicitly types `--pre-write` without `--intent` has a misconfiguration; silent exit-0 risks CI workflows marking the run green even though the pre-write step did not happen. Exit 2 is a distinct signal from the normal exit codes: 0 = all audit steps succeeded, 1 = audit step failed, 2 = audit succeeded but explicit `--pre-write` was requested and could not run.
- **Render extension** — new section `### Canonical drift` emitted by `_lib/pre-write-render.mjs`. Omitted when `advisory.drift` is empty. `CANONICAL DRIFT:` literal wording per `canonical/pre-write-gate.md` §5 example.
- **`tests/test-pre-write-integration.mjs`** — release-blocking end-to-end test. Multi-source fixture exercises all four intent lookup paths (name, file, dep, shape) AND canonical drift in one run. FP budget stays at 0.
- Three new test files plus two extended test files (drift, audit-repo-pre-write, integration are new; render and CLI are extended). Exact list in §6.
- `SKILL.md` — brief update to the `## Pre-write mode` section pointing at the `--pre-write` audit-repo flag.

## 4. Output shapes

### 4.1 `_lib/pre-write-drift.mjs`

```ts
computeDrift({ canonicalClaims, lookups }) → Array<{
  intentName: string,
  canonicalOwner: string,
  canonicalFile: string,
  canonicalLine: number,
  astOwners: string[],     // every AST owner observed, possibly empty
  kind: 'owner-disagrees' | 'ast-absent',
}>
```

- `owner-disagrees` — canonical declares X, AST observes Y (and not X). `astOwners` lists the Y set.
- `ast-absent` — canonical declares X, AST has zero identities. `astOwners` is `[]`.
- No drift entry is emitted for `aligned` or `not-consulted` — those are the healthy / irrelevant cases.

P1-3 drift is a read-only derivation — it does NOT re-parse canonical or re-run lookup. It consumes the P1-1 lookup results (which already carry `canonicalAstStatus`) and projects them into drift-entry shape.

### 4.3 CLI cold-cache flow

Artifact-by-artifact preflight. The table below is the authoritative list:

| Artifact (relative to `<output>/`) | Producer script | Failure kind | Timeout kind |
|---|---|---|---|
| `symbols.json` | `build-symbol-graph.mjs` | `cold-cache-symbols-failed` | `cold-cache-symbols-timeout` |
| `topology.json` | `measure-topology.mjs` | `cold-cache-topology-failed` | `cold-cache-topology-timeout` |
| `triage.json` | `triage-repo.mjs` | `cold-cache-triage-failed` | `cold-cache-triage-timeout` |

Flow:

```
pre-write.mjs --root . --output ./output --intent intent.json [--no-fresh-audit]

1. For each row in the table above:
   - If artifact exists → skip.
   - Else if --no-fresh-audit → record failures[{kind: '<artifact>-missing'}]; skip.
   - Else → spawn producer via execFileSync with argv array (NEVER a shell
           string). Timeout = COLD_CACHE_TIMEOUT_MS (60_000 by default).
       - On success: artifact exists; continue to the next row.
       - On non-zero exit: record failures[{kind: '<failure-kind>',
           stderr: e.stderr?.toString()?.slice(0, 400)}]; continue.
       - On timeout: record failures[{kind: '<timeout-kind>',
           timeoutMs: COLD_CACHE_TIMEOUT_MS}]; continue.

2. Load whatever artifacts now exist (some may still be missing if their
   producer failed or --no-fresh-audit was set).

3. Run name / file / dep / shape lookups. Missing artifact → lookup degrades
   to [확인 불가] rather than crashing.

4. Render / write advisory. The `capabilities` block copies whatever
   symbols.meta.supports ended up with. `failures[]` carries every cold-
   cache kind from step 1.
```

**Shell safety (P0):** every producer spawn MUST be `execFileSync(process.execPath, [scriptPath, '--root', root, '--output', output], opts)`. `execSync(\`node ${scriptPath} --root ${root}\`)` is forbidden — paths may contain spaces / `$` / `(` / `)`. Pinning test spins up a fixture under a path with spaces AND a `$` character; cold-cache must succeed.

**Stdout / stderr separation (P1):**
- `stdout` — Markdown advisory only. Exactly one stream of rendered sections.
- `stderr` — cold-cache producer logs + any `[pre-write] ...` diagnostic lines. Never interleaved with stdout.
Pinning test captures both streams and asserts the advisory-header pattern appears ONLY on stdout and producer log lines appear ONLY on stderr.

### 4.4 `audit-repo.mjs --pre-write` integration

- New opt-in flag `--pre-write`. When set, after the existing pipeline steps, run `pre-write.mjs --root <root> --output <output> --intent <intent-file>` via `execFileSync` (argv array — NO shell strings).
- `--intent <path>` required when `--pre-write` set. Missing-intent behavior:
  - Complete all other audit pipeline steps normally.
  - Write `manifest.json.preWrite = { requested: true, ran: false, reason: '--intent missing' }`.
  - Print `[audit-repo] --pre-write requested but skipped: --intent <file> missing` to stderr.
  - Exit with code **2** (distinct from 0 and 1 — signals "audit succeeded but an explicitly-requested pre-write did not run"). CI workflows that opt into `--pre-write` will treat exit 2 as failure by default; workflows that want to tolerate missing-intent can check the exit code explicitly.
- Successful pre-write: exit 0 (standard).
- NOT in default `quick` / `full` profiles. Callers opt in explicitly.

Exit code summary:

| Code | Meaning |
|---|---|
| 0 | All audit steps succeeded; pre-write either ran and succeeded, or was not requested. |
| 1 | An audit step failed (existing audit-repo semantic). |
| 2 | `--pre-write` requested but could not run due to missing `--intent`. |

## 8. Known risks

- **Cold-cache latency.** On a large repo, running all three producers sequentially may exceed pre-write-gate.md §4's 30-second cold-cache budget. Policy: `COLD_CACHE_TIMEOUT_MS = 60_000` per producer (constant at module top in `_lib/pre-write-cold-cache.mjs`, overridable via `PRE_WRITE_COLD_CACHE_TIMEOUT_MS` env var for tests). On timeout: record `failures[{kind: 'cold-cache-<artifact>-timeout', timeoutMs}]` and proceed. Downstream lookup for that artifact degrades to `[확인 불가, reason: cold-cache timeout]`. Does NOT parallelize producers — their ordering matters (topology depends on collect-files, etc.) and sequential `execFileSync` keeps the flow simple.
- **Cold-cache producer failures.** If `build-symbol-graph.mjs` crashes on the fixture repo (e.g. unresolvable tsconfig), the CLI must survive and report. Pinning in §5.3 case 4.
- **`topology.meta.complete` presence.** Integration test §5.6 assertion 8 (`NEW_FILE`) requires `topology.meta.complete === true`. The producer-side emission landed as a P1-2 preparatory patch in `measure-topology.mjs`; any rollback breaks integration. Bootstrap test (`tests/test-pre-write-bootstrap.mjs`) should grow an assertion that `topology.meta.complete === true` after a fresh `measure-topology.mjs` run.
- **Drift false positives.** When canonical is stale (e.g. canon-draft hasn't been refreshed after a refactor), every mismatch becomes a drift warning, flooding the advisory. P1-3 does NOT attempt to suppress these — the advisory is a flag, the user decides. If signal-to-noise becomes an issue, a `--max-drift-entries <N>` flag could be added in P2; not P1-3 scope.
- **`--pre-write` flag ordering.** Must come AFTER all other audit steps so its advisory reflects the freshest artifacts. If a user runs `--quick --pre-write`, pre-write runs against the quick-profile output (which omits some artifacts). Document this in SKILL.md; the advisory's `failures[]` already surfaces missing-artifact conditions.
- **Integration test fixture maintenance.** The fixture in §5.6 has a deliberately-missing `src/types/gone.ts` to force a drift entry. If a future refactor adds that path as a real file, drift disappears and the integration test fails — which is correct, it means the fixture needs a new deliberately-missing file to keep forcing drift.
- **Shell safety regression.** `execSync` anywhere in the new CLI or audit-repo code path is a P0 regression. The §5.3 case 5 pinning test exists specifically to catch this.

## 10. Hand-off to P2 (post-write)

After P1-3 ships, the full pre-write gate is:

1. `audit-repo.mjs --pre-write --intent intent.json` → runs the full pipeline + pre-write.
2. `pre-write-advisory.latest.json` + invocation-id JSON written; Markdown printed to stdout.
3. `advisory.drift[]` carries formal drift warnings.
4. `advisory.lookups[]` carries four kinds (name / file / dependency / shape) in stable order.
5. `advisory.capabilities` records what producers could emit at invocation time.

P2 consumes `pre-write-advisory.<invocationId>.json` as the "before snapshot" and produces a delta artifact:

- `post-write-delta.json` — planned vs observed `type-escape` comparison (any-contamination §6 Stage 2).

- `advisory.intent.plannedTypeEscapes` JSON order must remain caller-supplied (P1-1 pin).
