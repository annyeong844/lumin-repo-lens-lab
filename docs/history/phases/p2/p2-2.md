## 4. Output shapes

### 4.1 CLI flag contract (`audit-repo.mjs`)

```
audit-repo.mjs
  ... (existing flags) ...
  --pre-write                               (from P1-3)
  --intent <file>                           (required companion to --pre-write; from P1-3)
  --post-write                              (NEW — P2-2)
  --pre-write-advisory <file>               (required companion to --post-write; NEW — P2-2)
  --delta-out <dir>                         (forwarded to post-write.mjs; NEW — P2-2)
  --no-fresh-audit                          (forwarded; from P1-3)
  --include-tests / --no-include-tests      (forwarded to post-write.mjs)
  --production                              (forwarded)
  --exclude <pattern>                       (forwarded; multiple allowed)
```

**Mutual exclusion.** `--pre-write` and `--post-write` in the same invocation → exit 2 with stderr diagnostic. Rationale: the two modes operate on DIFFERENT lifecycle stages (before Claude writes vs after); combining them in one CLI call would produce a partially-baked advisory or a delta against a freshly-minted baseline (which is itself the post-state, defeating the point).

### 4.2 Exit-code contract (extended from P1-3)

| Code | Meaning |
|---|---|
| 0 | Audit succeeded; pre-write OR post-write (whichever was requested) ran and succeeded; OR neither was requested. |
| 1 | Existing audit-step-failed path (unchanged from pre-P2-2). |
| 2 | `--pre-write` requested without `--intent` (from P1-3), OR `--post-write` requested without `--pre-write-advisory` (new), OR `--pre-write` AND `--post-write` requested together (new). |

**`post-write.mjs` failures** (non-zero exit from the spawned child, e.g., unreadable advisory) → captured into `manifest.postWrite.reason` + final exit code stays 0. Rationale: same as P1-3 — the orchestrator's primary job is the audit; the opt-in post-write is advisory. Callers that need to gate on silent-new inspect `manifest.postWrite.silentNew` / `.requiredAcknowledgementCount` / `.ran`.

**Future `--strict-post-write`** (not in P2-2 v1): a later phase MAY add a flag that converts `manifest.postWrite.ran === false` into final exit 2, closing the "silent CI green on unreadable advisory" gap. P2-2 v1 leaves advisory semantics as the default and documents the gap in §8 risks.

### 4.3 `manifest.postWrite` block shape (with summary fields)

```ts
type ManifestPostWriteBlock = {
  requested: boolean,              // true iff --post-write flag was set
  ran: boolean,                    // true iff post-write.mjs spawned AND exited 0
  reason?: string,                 // populated only when !ran
  deltaPath?: string,              // absolute path to the ACTUAL latest artifact location — see below
  // Summary fields populated only when ran === true AND deltaPath parses cleanly.
  // Read from the delta JSON after the spawn; present to keep the manifest
  // self-describing so CI / caller-side Stage 3 gating does not require a
  // second JSON read.
  silentNew?: number,                      // = delta.summary.silentNew
  requiredAcknowledgementCount?: number,   // = entries.filter(label==='silent-new').length (always === silentNew by contract)
  baselineStatus?: 'available' | 'missing',
  scanRangeParity?: 'ok' | 'mismatch' | 'baseline-missing',
  afterComplete?: boolean,
};
```

**`deltaPath` resolution (reviewer P0-3 fix).** `deltaPath` points at the actual latest artifact path written by `post-write.mjs`. If `--delta-out <dir>` is supplied to `audit-repo.mjs`, it forwards to `post-write.mjs --delta-out`, and `post-write.mjs` writes there. In that case:

```
deltaPath = <delta-out>/post-write-delta.latest.json
```

Otherwise (default):

```
deltaPath = <output>/post-write-delta.latest.json
```

The orchestrator computes `deltaPath` by mirroring the same resolution `post-write.mjs` does: `delta-out || output` wins. Pinning test covers both branches.

**Summary field population.** After `execFileSync(post-write.mjs)` returns with exit 0, the orchestrator reads back the `deltaPath`:

```js
let summary = {};
try {
  const delta = JSON.parse(readFileSync(deltaPath, 'utf8'));
  summary = {
    silentNew: delta.summary?.silentNew ?? 0,
    requiredAcknowledgementCount: (delta.entries ?? []).filter((e) => e.label === 'silent-new').length,
    baselineStatus: delta.baseline?.status ?? 'missing',
    scanRangeParity: delta.scanRangeParity?.status ?? 'baseline-missing',
    afterComplete: delta.inventoryCompleteness?.afterComplete ?? false,
  };
} catch {
  // Spawn claimed success but artifact unreadable — leave summary fields empty.
  // `ran: true` + absent summary fields is the honest signal; downstream
  // treats missing fields as "unknown" rather than "clean".
}
postWriteBlock = { requested: true, ran: true, deltaPath, ...summary };
```

Populated states:

```
// --post-write not requested
manifest.postWrite = undefined  // field omitted from manifest JSON

// --post-write without --pre-write-advisory
manifest.postWrite = {
  requested: true,
  ran: false,
  reason: '--pre-write-advisory missing',
}
// + exit 2

// --pre-write AND --post-write together
manifest.postWrite = {
  requested: true,
  ran: false,
  reason: '--pre-write and --post-write are mutually exclusive',
}
// + exit 2 (manifest.preWrite block records symmetric reason)

// --post-write requested, spawn succeeded, delta readable
manifest.postWrite = {
  requested: true,
  ran: true,
  deltaPath: '/abs/.../post-write-delta.latest.json',
  silentNew: 2,
  requiredAcknowledgementCount: 2,
  baselineStatus: 'available',
  scanRangeParity: 'ok',
  afterComplete: false,
}

// --post-write requested, post-write.mjs exited non-zero
manifest.postWrite = {
  requested: true,
  ran: false,
  reason: 'post-write.mjs exited non-zero: <message>',
}
```

### 4.4 Forwarding semantics (reviewer P0-2 fix)

**Verbatim flag forwarding.** Each boolean flag set on `audit-repo.mjs` MUST also appear on the spawned `post-write.mjs` argv, including the negation form. The orchestrator does NOT translate `--production` into `--no-include-tests` or vice versa — both sides already parse both forms via `_lib/cli.mjs::parseCliArgs`.

```js
const forwardedArgs = [
  postWritePath,
  '--root', ROOT,
  '--output', OUT,
  '--pre-write-advisory', advisoryPath,
];
if (values['delta-out']) forwardedArgs.push('--delta-out', path.resolve(values['delta-out']));
if (values['no-fresh-audit'])       forwardedArgs.push('--no-fresh-audit');
if (values['include-tests'] === true)    forwardedArgs.push('--include-tests');
if (values['no-include-tests'] === true) forwardedArgs.push('--no-include-tests');  // <- reviewer P0-2
if (values['production'] === true)  forwardedArgs.push('--production');
for (const exc of (values['exclude'] ?? [])) forwardedArgs.push('--exclude', exc);

execFileSync(process.execPath, forwardedArgs, { stdio: ['ignore', 'inherit', 'inherit'] });
```

`stdio: 'inherit'` — post-write's Markdown stdout + `[post-write]` stderr diagnostics pass through the orchestrator transparently. Callers reading `audit-repo.mjs` stdout see audit summary first, then `## post-write delta` header + the Markdown body.

## 8. Known risks

- **Orchestrator exit code when post-write fails.** P1-3 precedent: pre-write non-zero exit does NOT crash the audit. P2-2 mirrors this. CI gating on silent-new must read `manifest.postWrite.silentNew` / `.requiredAcknowledgementCount` / `.ran`. A future `--strict-post-write` flag MAY convert `ran === false` into exit 2; P2-2 v1 does not implement it.
- **Mutual exclusion semantics.** `--pre-write` AND `--post-write` → exit 2, both manifest blocks populated. Alternative (chain them) rejected: the chained baseline would be the post-state, making silent-new detection meaningless.
- **Stdio inheritance.** Spawned `post-write.mjs` uses `stdio: ['ignore', 'inherit', 'inherit']`. Stdout interleaves orchestrator + delta Markdown, separated by `## post-write delta` header. Alternative (capture + echo) rejected: complicates streaming and creates buffer-size traps on >1MB deltas. Pinning test covers ordering + stderr segregation.
- **`occurrenceKey` collision in Fixture 1.** P2-0 risk: same (file, escapeKind, normalizedCodeShape, insideExportedIdentity) → same key → `typeEscapes[]` Map collapses. Fixture 1 uses TWO DIFFERENT files (`src/adapters/a.ts` + `src/adapters/b.ts`) to keep the two planned candidates distinct. If this design is ever "simplified" back to one file with two occurrences, the test will structurally fail because only one entry survives into the inventory — reviewer P0-4 fix preserved structurally by the fixture file-count.
- **Manifest summary fields present but delta unreadable.** If the spawn succeeds but the delta JSON fails to parse, `manifest.postWrite = {ran: true, deltaPath, /* summary fields omitted */}`. This is the honest signal — downstream treats missing fields as "unknown", NOT "clean". Pinning test writes a malformed delta to provoke this path.
